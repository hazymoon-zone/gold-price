# Gold Price Worker

A Cloudflare Worker that fetches gold prices on a schedule, stores the latest values in KV, and sends an email alert if the most recent price drops more than 10% compared to the previous value.

## Features

- Scheduled fetches via Cron Triggers.
- KV storage of price snapshots keyed by timestamp.
- Reduction check against the last two samples.
- Mailgun email alert when the drop exceeds 10%.

## Requirements

- Node.js and Bun
- Cloudflare Wrangler
- A KV namespace bound as `GOLD_PRICE`
- Mailgun API key and domain

## Setup

Install dependencies:

```bash
bun install
```

Generate Worker types (after changing bindings in `wrangler.jsonc`):

```bash
npx wrangler types
```

## Configuration

`wrangler.jsonc` defines:

- `triggers.crons`: the schedule (runs in UTC)
- KV binding: `GOLD_PRICE`
- compatibility flags (Node.js compatibility is enabled)

### Environment Variables (Secrets)

Set these as Worker secrets:

- `MAILGUN_API_KEY`
- `MAILGUN_SANDBOX` (the Mailgun domain)

Add secrets:

```bash
npx wrangler secret put MAILGUN_API_KEY
npx wrangler secret put MAILGUN_SANDBOX
```

## Development

Run locally with the scheduled handler enabled:

```bash
npx wrangler dev --test-scheduled
```

Trigger the scheduled handler manually:

```bash
curl "http://localhost:8787/__scheduled?cron=*+*+*+*+*"
```

## Deploy

```bash
npx wrangler deploy
```

## How It Works

1. `updateGoldPrice` fetches the latest data and stores the selected price in KV.
2. `getRecentGoldPrices` loads the two most recent KV entries.
3. `checkHitLimitReduction` compares them and checks for a >10% drop.
4. `sendAlertEmail` posts to Mailgun to send an alert.

## Notes

- Cron triggers are evaluated in UTC by Cloudflare.
- `worker-configuration.d.ts` is generated by `wrangler types`.
