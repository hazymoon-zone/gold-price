# Gold Price Worker

A Cloudflare Worker that fetches gold prices on a schedule, stores snapshots in Workers KV, and sends an alert email when a downward move reaches the configured threshold.

## Features

- Scheduled fetches using Cron Triggers.
- KV storage keyed by ISO timestamp (7-day TTL per snapshot).
- Downtrend detection across recent unique prices.
- Alert email via Mailgun when price reduction is >= 1%.
- Duplicate-alert protection using the `most-recent-key` marker in KV.

## Requirements

- Bun
- Cloudflare account + Wrangler CLI access
- A KV namespace bound as `GOLD_PRICE`
- Mailgun API key and sandbox domain

## Setup

Install dependencies:

```bash
bun install
```

Generate Worker runtime types (run this after changing bindings in `wrangler.jsonc`):

```bash
bun run cf-typegen
```

## Configuration

`wrangler.jsonc` currently defines:

- `main`: `src/index.ts`
- `triggers.crons`: `0 1-23/2 * * *`
- `kv_namespaces`: binding `GOLD_PRICE`
- `compatibility_flags`: `nodejs_compat`
- `vars.MAILGUN_SANDBOX`: Mailgun domain used by email sending

### Secrets and Environment Variables

Required secret:

- `MAILGUN_API_KEY`

Mailgun domain:

- `MAILGUN_SANDBOX` is currently configured in `wrangler.jsonc` via `vars`.
- You can also store it as a secret if you do not want it in config.

Set secret values:

```bash
bunx wrangler secret put MAILGUN_API_KEY
# Optional if you want to move domain from vars to secret:
bunx wrangler secret put MAILGUN_SANDBOX
```

## How It Works

1. `updateGoldPrice` fetches latest prices, filters target item data, and writes unique price points to KV with a 7-day TTL.
2. The worker lists KV keys, skips the `most-recent-key` marker, and ignores processing when there is no new data.
3. `getGoldPriceDownTrend` builds a downward trend sequence from recent unique values.
4. `checkHitLimitReduction` calculates percentage reduction and checks whether it is >= 1%.
5. `sendAlertEmail` sends a Mailgun message containing previous/current prices and reduction percentage.
6. `most-recent-key` is updated in a `finally` block to avoid duplicate alerts for the same latest snapshot.

## Notes

- Cron expressions are evaluated in UTC by Cloudflare.
- Comment in `wrangler.jsonc` indicates local business-time intent (UTC+7), but actual schedule execution is UTC.
- `worker-configuration.d.ts` is generated by Wrangler (`bun run cf-typegen`).
